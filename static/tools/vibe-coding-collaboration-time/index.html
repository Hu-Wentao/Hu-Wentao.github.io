三种协作模式耗时模拟器

三种协作模式使用完全相同的 SPEC、沟通轮数和单位耗时，只改变任务的调度方式。结果是模型估算，不包含最终代码落地和运行检查。

任务与资源

所有时间单位均为分钟。AI 时间指一个 SPEC 在一轮中的分析与修改时间。

平均返工率

返工率表示一个 SPEC 在首轮后仍需再沟通一次的平均概率。无需指定返工发生在哪个 SPEC，模拟结果会对所有可能组合加权取平均。

查看公式、调度策略与边界

令 N 为 SPEC 数，p 为平均返工率，e₁/e₂ 为首轮/返工编辑时间，a 为单个 AI 任务时间，v 为审查时间。

平均返工 SPEC = Np；平均 SPEC-round = N(1 + p) 即时：E[T] = N(e₁ + a + v) + Np(e₂ + a + v) 邮件与文档：对每个 SPEC 独立以概率 p 返工一次，并对所有组合的调度结果加权取平均

文档协作先连续完成所有首轮编辑，同时把修改交给最早空闲的 AI；之后优先审查最早完成的结果，需要返工时立即编辑并再次提交，不等待其他 SPEC 组成整批。

  • 开发者只有一个执行槽，编辑和审查不能并行。
  • AI 任务相互独立、耗时相同，且不限制同时处理的任务数量。
  • 每个 SPEC 最多返工一次；三种协作模式使用相同的返工概率和期望工作量。
  • 不计交接、共享文件冲突、最终代码落地与运行检查。

该工具比较的是 SPEC 协作阶段的确定性调度结果，而不是对所有团队和任务都成立的性能承诺。
