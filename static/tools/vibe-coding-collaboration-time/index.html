三种沟通模式耗时模拟器

任务与资源

所有时间单位均为分钟。AI 时间指一个 SPEC 在一轮中的分析与修改时间。

查看公式、调度策略与边界

令 nᵣ 为第 r 轮仍需沟通的 SPEC 数，eᵣ 为该轮编辑时间，a 为单个 AI 任务时间，v 为审查时间，C 为 AI 并发数。

即时：T = Σᵣ nᵣ(eᵣ + a + v) 邮件：T = Σᵣ [nᵣ(eᵣ + v) + ⌈nᵣ / C⌉a] 文档：makespan{ 编辑ᵢ,ᵣ → AIᵢ,ᵣ → 审查ᵢ,ᵣ → 下一轮编辑ᵢ,ᵣ₊₁ }

文档模式先连续完成所有首轮编辑，同时把修改交给最早空闲的 AI；之后优先审查最早完成的结果，需要返工时立即编辑并再次提交，不等待其他 SPEC 组成整批。

  • 开发者只有一个执行槽，编辑和审查不能并行。
  • AI 任务相互独立、耗时相同，共用 C 个执行槽。
  • 三种模式使用相同的逐 SPEC 轮数；返工位置只影响文档模式的关键路径。
  • 不计交接、共享文件冲突、最终代码落地与运行检查。

该工具比较的是 SPEC 协作阶段的确定性调度结果，而不是对所有团队和任务都成立的性能承诺。
